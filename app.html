Troubleshooting Remote VPN Connectivity Drops

PH
Prabhnoorjit Hans
Published: Aug 12, 2026
Updated: 2 weeks ago
Views: 1,402
Visibility: Public

  • 2. Resolution Steps (Windows 11)

1. Overview & Scope

This guide addresses the common issue where end users experience frequent disconnects from the corporate VPN, specifically when utilizing split-tunneling on Windows 11 machines connecting via home ISPs.

2. Resolution Steps

Step 1: Verify the client version.
Ensure the user is running the latest version of the VPN client (v4.2.1 or higher). Older versions have known compatibility issues with recent Windows OS updates.

Step 2: Check MTU Settings.
Often, home ISP routers fragment packets if the MTU is too high. Instruct the user or use remote management to lower the VPN adapter MTU to 1350.

netsh interface ipv4 set subinterface "VPN Connection" mtu=1350 store=persistent

Step 3: Update Network Drivers.
Check the linked hardware resource in ServiceLink. If the device is a Dell XPS model from 2025, push the latest Intel Wi-Fi driver update via the management console.

3. Escalation

If the issue persists after these steps, escalate the ticket to the Level 2 Network team and attach this KB article indicating steps already taken.

Resolving Zoom Audio Issues on Mac

AM
Arshdeep Mutti
Published: Mar 22, 2026
Visibility: Public

Symptom

Users on macOS Sonoma frequently report that Zoom cannot capture audio from their microphone, even if the correct microphone is selected in Zoom settings.

Resolution

Step 1: Have the user open System Settings > Privacy & Security > Microphone.

Step 2: Ensure the toggle next to zoom.us is turned ON.

Step 3: If the toggle is missing, the user must reinstall Zoom to trigger the system prompt for microphone access again.

Step 4: Check the "Core Audio" service if the above fails by running sudo killall coreaudiod in the Terminal to reset the macOS audio daemon.
